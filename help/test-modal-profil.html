🔧 Tests à Effectuer (depuis le dashboard)

Prérequis: Être connecté au dashboard en tant qu'utilisateur

  1. Ouvrir la modal: Cliquez sur votre photo/profil en haut à droite
  2. Mode édition: Cliquez sur le bouton "Modifier"
  3. Modifier les informations:
    • Changez le prénom
    • Changez le nom
    • Changez le téléphone
    • Changez l'email
  4. Changer la photo:
    • Cliquez sur "Changer la photo"
    • Sélectionnez une image depuis votre ordinateur
    • Vérifiez que la preview s'affiche
  5. Changer le mot de passe:
    • Entrez votre mot de passe actuel
    • Entrez un nouveau mot de passe (min 6 caractères)
    • Confirmez le nouveau mot de passe
  6. Visibilité du mot de passe: Cliquez sur l'icône oeil pour afficher/masquer
  7. Sauvegarder: Cliquez sur "Sauvegarder"
  8. Réinitialisation: Cliquez sur le bouton rouge "Réinitialiser" (danger)

📡 APIs Implémentées

Mise à jour du profil

Endpoint: POST /backend/Api/Auth/update-profile.php

Données:

{
  "prenom": "John",
  "nom": "Doe",
  "telephone": "0123456789",
  "email": "john@example.com",
  "password": "newPassword123",  // optionnel
  "oldPassword": "currentPassword" // optionnel
}

Upload de photo

Endpoint: POST /backend/Api/Auth/upload-photo.php

Données: multipart/form-data avec fichier 'photo'

Restrictions:

  • Type: JPEG, PNG, GIF, WebP
  • Taille max: 5MB
  • Dossier de destination: /backend/uploads/

📝 Notes d'Implémentation

  • Les champs sont désactivés (readonly) par défaut
  • Le bouton "Modifier" bascule l'état éditable
  • Les modifications sont validées côté client et serveur
  • La photo est stockée dans /backend/uploads/profiles/
  • L'utilisateur est mis à jour dans localStorage après sauvegarde
  • Le header est mis à jour immédiatement après modification
  • Les données sont envoyées en JSON via FETCH API
  • La réinitialisation est maintenant dans la modal profil (pas en sidebar)
